# DMS panel config - placeholder

DMS (Dank Material Shell) isn't installed anywhere I can introspect from this
session, so this directory is a stub rather than a real config.

When you have DMS installed on the CachyOS VM:

1. Run DMS once so it generates its default config, then paste its path/contents
   back so we can build the real config here (module layout matching the
   waybar set: workspaces left, clock/status center, tray/bluetooth/network/
   audio/battery right - see the omarchy-configs waybar reference at
   `config/home/pamac/.config/waybar/config.jsonc` for the module list to
   replicate).
2. Wire DMS's color source to `~/.config/cachy-dots/current/theme/dms-colors.json`
   (generated by `theme-engine/theme-set.sh` from `theme-engine/templates/dms-colors.json.tpl`)
   instead of its default Matugen wallpaper-driven theming.
3. Set the panel font to JetBrainsMono Nerd Font, matching the rest of this setup.
4. Turn off/ignore DMS's own launcher and notification center - Walker and mako
   stay the single source of truth for launching and notifications.
5. Update `config/systemd/user/dms.service`'s `ExecStart=` to the real DMS binary
   name/path once known (it's currently a placeholder guess of `/usr/bin/dms`).
6. Set your preferred Niri keybinds through DMS's own settings panel - that's
   explicitly out of scope for this repo (see the plan doc for why).
